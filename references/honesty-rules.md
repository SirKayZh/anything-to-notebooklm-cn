# 诚实度规范（Honesty Rules）

> 本 Skill 的核心立身之本。所有 scenarios 输出前必读。
>
> **设计原则**：宁可写得平实，不可写得"以假乱真"。

---

## 为什么这件事重要

AI 内容工具最大的伦理漏洞是：**让用户以为 AI 看过/听过/读过原始内容**，
但实际上 AI 只是基于元数据/标题/部分内容做了"风格化总结"。

用户基于这种"伪完整"的输出做决策（推荐给朋友、写成文章、做投资判断），
风险被默默转嫁到用户身上。

本 Skill 拒绝这种模式。**输出必须如实标注信息来源置信度**。

---

## 1. 三级信息来源分类

| 级别 | 标记 | 含义 | 示例 |
|---|---|---|---|
| **L1 事实** | 直陈 | 原文/元数据中明确表述 | "节目 78 分钟" / "作者：产品欧sir" |
| **L2 引用** | 「」包裹 | 直接抄原文金句 | 「我们真是最后一代古代人」 |
| **L3 推断** | *(推断)* | 基于已知信息的合理推测 | "本期重点是 AI 时代焦虑感 *(推断)*" |
| **L4 建议** | 加注释 | AI 提供的额外建议/评价 | "推荐讲法：…… *(AI 建议)*" |

L1 / L2 是"真"，L3 / L4 是"AI 加工"。读者必须能一眼分清。

---

## 2. 抓取完整度声明

每份输出**顶部** 必带：

```markdown
> **抓取完整度**: full | partial | metadata-only
> **未抓到**: 音频内容 / 付费正文 / 评论区 / ……
```

具体场景对照：

| 场景 | 完整度 | 警告 |
|---|---|---|
| 公众号文章 | partial | 「本分析基于 WebFetch 抓到的正文，可能不是完整原文」 |
| 小宇宙节目 | metadata-only | 「⚠️ 基于节目页章节，未听原音频」|
| 知识星球 | full（用户粘贴） | 无 |
| 财新（无 cookie） | metadata-only | 「⚠️ 仅获取到摘要部分」|
| B站视频 | metadata-only | 「基于章节标题，未观看视频」|

---

## 3. NotebookLM 生成内容标注

NotebookLM 的 Audio Overview / Mind Map / Briefing Doc 都会**演绎**：

```markdown
🤖 以下内容由 NotebookLM 基于 source 演绎生成。
   可能包含：
   - 未在原文出现的过渡语和判断
   - 把单一观点扩展为两人辩论
   - 来自 NotebookLM 训练数据的背景知识

   原始 source: [link]
```

特别是 Audio Overview 生成的"双人对话"，**不是嘉宾原话**，是 AI 演绎的虚拟主播。

---

## 4. 输出 metadata 强制 schema

所有产物（mindmap / summary / analysis-report 等）**末尾**必带：

```json
{
  "source_url": "https://...",
  "fetched_at": "2026-05-20T18:30:00+08:00",
  "fetcher": "WebFetch / yt-dlp / wexin-read MCP / ...",
  "source_completeness": "full | partial | metadata-only",
  "ai_inference_ratio": "low | medium | high",
  "method": "C 方案 LLM 12 问 / NotebookLM Audio Overview / ...",
  "warnings": [
    "未听原音频，摘要基于章节标题",
    "未读完整正文"
  ]
}
```

`ai_inference_ratio` 判定规则：

| 评分 | 标准 |
|---|---|
| **low** | 输出 90%+ 来自原文事实+引用，AI 加工 < 10% |
| **medium** | 输出 50-80% 来自原文，其余是合理推断 |
| **high** | 输出 < 50% 来自原文（仅元数据），AI 演绎为主 |

---

## 5. 推断的标注示例

### ❌ 不诚实的写法

```markdown
## 核心观点

1. 作者认为 GPTs 流量 = 价值
2. 这种思路忽视了商业化路径
```

→ 这两句都是 AI 推断，但用陈述句写得像原文事实。

### ✅ 诚实的写法

```markdown
## 核心观点（基于章节标题推断）

1. 作者似乎默认 GPTs 流量 = 价值 *(推断，原文未明说)*
2. AI 反思：这个论点忽视了商业化路径 *(AI 评价)*
```

---

## 6. 输出底部强制声明（模板）

每份输出底部加：

```markdown
---

## 信息来源声明

- **原始来源**：[URL]
- **抓取方式**：[WebFetch / yt-dlp / 用户粘贴 / ...]
- **完整度**：[full / partial / metadata-only]
- **AI 演绎程度**：[low / medium / high]
- **未覆盖的信息**：[列出未抓到/未听到的部分]

⚠️ 本输出包含 AI 推断和建议，请以原始来源为准。
```

---

## 7. 自查清单

写完每份输出，回头检查：

- [ ] 三类内容（事实/引用/推断）是否都有明确标记？
- [ ] 顶部是否声明了抓取完整度？
- [ ] 是否避免了"AI 看过原始内容"的语气？
- [ ] 底部是否有 metadata + 来源声明？
- [ ] confidence 字段是否如实填写？

## 8. 与上游 (joeseesun) 的差异

| 维度 | 上游 | 本 Skill |
|---|---|---|
| 抓取宣传 | "6 层级联绕过 300+ 付费墙" | "先看再决定，标注完整度" |
| 输出宣传 | "AI 自动生成播客" | "AI 演绎，标注来源" |
| 用户预期 | "这是完整的内容" | "这是基于 X 的 AI 加工" |

**本 Skill 的卖点不是"抓得更猛"，而是"用得更安心"。**
